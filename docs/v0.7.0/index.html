clawdcursor doctor clawdcursor start POST /task

clawdcursor mcp POST /execute/{tool} Claude Code Cursor

Six Layers

Cheap layers first, vision last

Each task falls to the cheapest layer that can handle it. Vision is the last resort — not the first call. Only active in CLI agent mode.

⚡

L1 · Action Router — $0

Pattern-matched tasks: open apps, type text, press shortcuts, navigate URLs. Zero LLM calls. Instant execution.

🔍

L2 · A11y + CDP Reasoner — text model

Reads the accessibility tree and Chrome DevTools Protocol. Text model reasons about what to interact with. Bounds-based click — no vision needed.

🌐

L0 · Browser / Playwright

Handles browser-specific tasks first via Playwright automation before escalating to the full pipeline.

🎯

L3 · Computer Use — vision model

Full vision model with screenshot + action loop. Last resort for anything L1 and L2 can't handle. Supports any visible UI.

What's New

Built for reliability, not just features

v0.7.0 fixes the core failure modes that made v0.6.3 frustrating in practice.

✅

Ground-Truth Verifier

Reads actual screen state after every action. Blocks false "done" reports — the agent can't lie about success anymore.

🔁

Anti-Loop Detectors

No-progress detector blocks after 3 identical failed actions. Premature-done blocker requires evidence before accepting task completion.

🌐

CDP Browser Control

Chrome DevTools Protocol lets clawd-cursor click by CSS selector, read DOM text, and type into labeled fields — no vision model needed for web tasks.

🔌

Native MCP Support

Plug directly into Claude Code, Cursor, Windsurf, or Zed as an MCP server. No REST plumbing — tools show up natively in your AI's registry.

🤖

Truly Model-Agnostic

Text and vision models can come from different providers. Groq for L2 speed, Anthropic for L3 vision. Auto-detected by clawdcursor doctor.

🏠

Standalone — No Platform Lock

Decoupled from OpenClaw. Data lives in ~/.clawd-cursor/. Auto-migrates from legacy paths. Works with any client.

Get Started

Up and running in 2 minutes

Install, consent once, connect your AI.

Terminal + claude_desktop_config.json
# 1. Install
npm install -g clawd-cursor

# 2. One-time consent (required — desktop control warning)
clawdcursor consent

# 3. Add to your MCP client config
{
  "mcpServers": {
    "clawd-cursor": {
      "command": "clawdcursor",
      "args": ["mcp"]
    }
  }
}

# clawd-cursor mcp will refuse to start without consent
Terminal
# 1. Install
npm install -g clawd-cursor

# 2. Auto-detect your AI providers (Anthropic, OpenAI, Ollama, etc.)
clawdcursor doctor

# 3. Start the agent (consent prompt fires on first run)
clawdcursor start

# 4. Send a task
clawdcursor task "Open Chrome and go to github.com"

# Or open an interactive task console
clawdcursor task
Terminal + HTTP
# 1. Install and start
npm install -g clawd-cursor
clawdcursor start   # consent fires on first run

# 2. List all 33 tools + schemas
GET  http://localhost:3847/tools

# 3. Call a tool directly
POST http://localhost:3847/execute/desktop_screenshot
POST http://localhost:3847/execute/mouse_click
     { "x": 450, "y": 300 }

# 4. Or send a plain-English task to the agent
POST http://localhost:3847/task
     { "task": "Open Gmail and read latest email" }

# Non-interactive consent for CI / scripted environments
clawdcursor consent --accept

Requires Node.js 20+. Consent is one-time and stored in ~/.clawd-cursor/consent. Server binds to localhost only.
